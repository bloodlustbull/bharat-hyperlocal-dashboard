# Evidence RAG Backend Contract

This dashboard is static, so real scraping and AI generation must run through a backend.
Do not place API keys in browser code — use `backend/.env.local` instead.

## Quick Start (Free, Local)

```bash
# Terminal 1: Start the RAG backend
npm run rag:server
# → http://127.0.0.1:8787/api/rag

# Terminal 2: Start the frontend
npm run dev
# → http://127.0.0.1:5500
```

Then in the Evidence RAG tab:
1. Set **Backend URL** to `http://127.0.0.1:8787` and click **Save**
2. Click **Generate AI evidence brief** — works immediately with deterministic fallback

### Optional: Enable AI-generated briefs (free with Ollama)

1. Install [Ollama](https://ollama.com)
2. Pull a model: `ollama pull llama3.1:8b`
3. Ensure Ollama is running (`http://127.0.0.1:11434`)

The backend auto-detects Ollama. No `.env` edits needed.

### Optional: Live web ingestion

Sign up for [Tavily](https://tavily.com) (free tier) and add to `backend/.env.local`:

```
TAVILY_API_KEY=tvly-your-key-here
```

## Configuration

### `.env.local` (recommended — never committed)

```env
# Ollama (free, local) — for AI brief generation
OLLAMA_ENABLED=true
OLLAMA_MODEL=llama3.1:8b
OLLAMA_BASE_URL=http://127.0.0.1:11434

# Default provider: "ollama" (free) or "openai" (paid key)
LLM_PROVIDER=ollama

# Tavily (free tier available) — for live web search
# TAVILY_API_KEY=tvly-...

# Server port
RAG_PORT=8787
CORS_ORIGIN=http://127.0.0.1:5500
```

### `.env` (fallback — can be committed with safe defaults)

Only non-secret defaults go here. The server loads `.env.local` first, then `.env`.

## Frontend Configuration

Set the backend URL in the Evidence RAG tab. It saves to:

```
localStorage.bharatRagApiUrl
```

## Endpoints

### GET `/health` or `/api/rag/health`

Returns provider status, LLM config, and key counts.

### POST `/brief` or `/api/rag/brief`

Generate a citation-first AI brief.

**Request:**
```json
{
  "context": {
    "city": "Hyderabad",
    "language": "Telugu",
    "category": "Snacks",
    "platform": "Blinkit",
    "objective": "First order",
    "channel": "WhatsApp broadcast"
  },
  "job": { "id": "hyd_snacks_market_signals", "title": "Hyderabad Snacks Market Signals" },
  "promptTemplate": {},
  "localEvidence": { "marketSources": [], "marketSignals": [], "sampleEvidence": [] }
}
```

**Response modes:**
- `backend_ollama` — generated by Ollama (free, local)
- `backend_deterministic` — rule-based fallback (no AI needed, always works)
- `backend_ai_error` — AI failed, includes error + fallback sections

### POST `/ingest` or `/api/rag/ingest`

Trigger live evidence ingestion.

### POST `/assistant` or `/api/assistant`

Voice assistant intent routing and responses (used by `voice-assistant.js`).

### GET `/evidence` or `/api/rag/evidence`

Returns all stored evidence records.

## Evidence Record Schema

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique slug-based identifier |
| `city` | string | Market city |
| `category` | string | Product category |
| `source_type` | string | tavily_search, page_fetch, market_signal, etc. |
| `source_name` | string | Human-readable source name |
| `source_url` | string | Original URL (if available) |
| `captured_at` | ISO string | When the evidence was captured |
| `claim` | string | One-line claim statement |
| `evidence_text` | string | Raw evidence excerpt (≤1200 chars) |
| `confidence` | string | A (primary/official), B (credible), C (directional), D (assumption) |
| `needs_validation` | boolean | True for unverified evidence |
| `tags` | string[] | Categorization tags |

## Confidence Rules

- **A**: Primary source, company filing, official API, or platform-owned public data
- **B**: Credible third-party source, established news/reporting, or reputable data provider
- **C**: Directional social/forum/creator signal or vendor/case-study source
- **D**: Manual observation, incomplete data, or assumption requiring validation
